Met à jour les informations d’un utilisateur (nom d’utilisateur, email).

Route : PUT /users/{uuid}/update

Paramètres :

  • uuid : L’ID de l’utilisateur à mettre à jour (dans le chemin de la requête).
  • Payload JSON : Contient les champs username et email (optionnels).

Réponse :

  • Succès (200 OK) : Un message confirmant que l’utilisateur a été mis à jour.
    "User updated"
  • Erreur (500 Internal Server Error) : Si la mise à jour échoue.
    "Update failed"
